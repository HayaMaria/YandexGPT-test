### Использование сервиса и платёжные карты.
Вы можете пользоваться сервисом без привязки платёжной карты. В этом случае реальные 
деньги не будут списываться, а сервис работает в пределах бесплатного лимита запросов. 
Любое «потребление» фиксируется в системе для статистики, но пока карта не привязана, 
это не создаёт обязательств к оплате.

Если вы привяжете карту, с неё будут списаны средства за все ресурсы, использованные
с момента создания аккаунта, а также за будущие запросы. Сумма списания зависит от количества 
и типа запросов, сделанных через сервис, и система автоматически списывает только то, 
что соответствует вашему фактическому потреблению.


### Инструкция для получения ID фолдера и JSON-файла с ключом в Yandex Cloud Console.
1. Создание фолдера.
- Перейдите в https://console.yandex.cloud. 
- Выберите своё облако или создайте новое. 
- Нажмите «Создать каталог» или «Создать фолдер»(кнопка справа на экране или через троеточие рядом с именем облака).
- После создания имя фолдера появится под именем облака. Войдите в него, если это не произошло автоматически.

2. Получение ID фолдера.
- Справа от имени фолдера есть кнопка «копировать».
- Или в адресной строке браузера (в URL после /folders/). 
  - Например: https://console.yandex.cloud/folders/b1g82u1r09dr8pd
  - ID фолдера - b1g82u1r09dr8pd

3. Создание сервисного аккаунта.
- Убедитесь, что находитесь внутри нужного фолдера (его имя указано вверху).
- Выберете: «Сервисные аккаунты» → «Создать сервисный аккаунт», воспользовавшись панелью под названием фолдера, 
либо нажав на троеточие рядом с именем фолдера.
- После этого появится новая строка в списке аккаунтов.

4. Назначение роли -- ai.languageModels.user.
- Откройте созданный фолдер. 
- Перейдите на вкладку «Управление доступом» или «Права доступа», воспользовавшись панелью под названием фолдера, 
либо нажав на троеточие рядом с именем фолдера. 
- Нажмите «Настройки доступа» → выберите свой созданный ранее сервисный аккаунт → Нажмите «Добавить роль» → 
Выберете «ai.languageModels(user)». 
- Нажмите «Сохранить». 

Теперь сервисный аккаунт имеет право вызывать YandexGPT и другие языковые модели.

5. Создание авторизационного ключа.
- Перейдите в «Сервисные аккаунты» (внутри фолдера). 
- Кликните по своему аккаунту. 
- На панели сверху «Создать новый ключ». 
- Выберите тип ключа: Авторизационный ключ (или Key pair).

**Не путайте с API-ключом — нужен именно авторизационный!**

- Нажмите «Создать».
- После этого скачайте JSON-файл с ключом, если это не произошло автоматически.

**Сохраните его в надёжном месте — повторно скачать его будет невозможно.**

- JSON-файл должен выглядеть как-то так:

  - { 
  - "id": "aje7tl2r2947jno11", 
  - "service_account_id": "ajegud8bh9kka657", 
  - "created_at": "2025-11-05T10:00:08.747421980Z", 
  - "key_algorithm": "RSA_2048", 
  - "public_key": "-----BEGIN PUBLIC KEY-----\длинная_строка-----END PUBLIC KEY-----\n", 
  - "private_key": "PLEASE DO NOT REMOVE THIS LINE! Yandex.Cloud SA Key ID <aje7tl2r2947jno10cf8>\n-----BEGIN 
  PRIVATE KEY-----\ОЧЕНЬ_длинная_строка==\n-----END PRIVATE KEY-----\n" 
  - }

6. Сохранение данных.
- ID фолдера → ID_folder.txt 
- JSON-файл → secrets.json

**Примечание:** ID фолдера может начать работать не сразу после создания.
